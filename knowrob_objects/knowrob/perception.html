perception.pl -- Asserting and reasoning about perception events.

These events allow temporal representation of the object pose via reification: VisualPerception events may have eventOccursAt properties that are used for computing the pose property of an object.

create_joint_information(+Type, +Parent, +Child, +Pose, +Direction, +Radius, +Qmin, +Qmax, -Joint) is det
Create a joint of class Type at pose Pose, linking Parent and Child Qmin and Qmax are joint limits as used in the ROS articulation stack

Usage: create_joint_information('HingedJoint', knowrob:'cupboard1', knowrob:'door1', [1,0,0,...], [], 0.23, 0.42, -Joint) create_joint_information('PrismaticJoint', knowrob:'cupboard1', knowrob:'drawer1', [1,0,0,...], [1,0,0], 0.23, 0.42, -Joint)

Arguments:
Type- Type of the joint instance (knowrob:HingedJoint or knowrob:PrismaticJoint)
Parent- Parent object instance (e.g. cupboard)
Child- Child object instance (e.g. door)
Pose- Pose matrix of the joint as list float[16]
Direction- Direction vector of the joint. float[3] for prismatic joints, [] for rotational joints
Radius- Radius of a rotational joint
Qmin- Minimal configuration value (joint limit)
Qmax- Minimal configuration value (joint limit)
Joint- Joint instance that has been created
update_joint_information(+Joint, +Type, +Pose, +Direction, +Radius, +Qmin, +Qmax)
Update type, pose and articulation information for a joint after creation. Leaves Parent and Child untouched, i.e. assumes that only the estimated joint parameters have changed.

Direction- Direction vector of the joint. float[3] for prismatic joints, [] for rotational joints
Radius- Radius of a rotational joint
Qmin- Minimal configuration value (joint limit)
