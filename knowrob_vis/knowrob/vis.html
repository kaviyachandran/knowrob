show(+VisualThing, +Instant, +Properties) is det
VisualThing is a thing with a visual interpretation and some way to generate a ROS visualization message for it. This includes marker_visualization messages and data_vis messages. VisualThing may be a RDF IRI of some OWL individual, a marker term (e.g., object(Iri)), or a data vis object (e.g., timeline(Identifier)). All existing markers are updated for the current timepoint if VisualThing is left unspecified. Properties is a list of properties passed to the respective submodules (i.e., marker or data visualization). If VisualThing is a list then each element is expected to be a term describing one visualization artifact.

camera_pose(+Position:list, +Orientation:list) is det
Sends a pose via the ROS topic /camera/pose. Visualization clients may choose to manipulate some 3D camera accordingly.
Arguments:
Position- [float x,y,z]
Orientation- [float qx,qy,qz,qw]
visualisation_server is det
Launches a web server that runs a minimal visualization client. This implementation is able to handle most marker visualization messages, data visualization messages (among other features) are unsupported. A proper client is implemented in openEASE. You can access the minimal client via http://localhost:1111
